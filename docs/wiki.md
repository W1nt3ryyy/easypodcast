# Общая архитектура и принцип работы EasyPodcasts

EasyPodcasts — это полноценное веб-приложение для добавления и прослушивания RSS подкастов.
Проект разделен на несколько логических слоев, каждый из которых работает в изолированном Docker-контейнере.

## Стек технологий
1. **Фронтенд:** React, собранный с помощью Vite. Интерфейс стилизован кастомным CSS (Glassmorphism, градиенты, современные шрифты).
2. **Бэкенд:** Django, который служит API-сервером. Использует библиотеку `feedparser` для извлечения информации из внешних источников (RSS).
3. **База данных:** PostgreSQL 18. Сохраняет подкасты и закладки.
4. **Веб-сервер/Прокси:** Apache (httpd). Раздает статичные файлы фронтенда и маршрутизирует запросы (через `ProxyPass`) на `/api/` к Django.

---

## Контейнеры (сервисы)
В `docker-compose.yml` описано 3 основных сервиса:
- `db`: База данных PostgreSQL. Данные хранятся локально в Docker томе `postgres_data`.
- `django-web`: Образ на базе Python. Выполняет автоматические миграции и поднимает внутренний сервер разработчика (через `manage.py runserver 0.0.0.0:8000`).
- `frontend`: Образ на базе `httpd:alpine`, в который перенесена собранная production-версия React-кода из промежуточного Node-контейнера. Выступает прокси-шлюзом для всего сайта.

Для разработки доступны дополнительные сервисы (запускаются с профилем `dev`):
- `django-dev`: Django с hot-reload для бэкенда.
- `frontend-dev`: Vite dev-сервер с hot-reload для фронтенда.

## Режимы запуска

### Production
```bash
docker compose up -d
```
- Фронтенд: http://localhost:80 (Apache, production build)
- Django API: http://localhost:8001

### Development (с hot-reload)
```bash
docker compose --profile dev up -d
```
- Фронтенд: http://localhost:5173 (Vite dev server)
- Django API: http://localhost:8000 (через Vite proxy)

| Сервис | Production | Development |
|--------|------------|-------------|
| Frontend | :80 (Apache) | :5173 (Vite) |
| Django | :8001 | :8000 (прокси) |
| Hot-reload | ❌ | ✅ |

## Как запустить проект с нуля

Для запуска вам понадобится только установленный Docker и Docker Compose на вашем устройстве.

1. Убедитесь, что Docker Desktop или Docker Engine запущен.
2. В корневой директории проекта выполните команду:
   ```bash
   docker compose up --build -d
   ```
   Эта команда скачает нужные образы, соберет зависимости (Pip для Python, NPM для JS) и запустит контейнеры в фоне.
3. Откройте браузер по адресу [http://localhost/](http://localhost/). (Порт 80).
4. Готово! Вы можете начать добавлять RSS ссылки в библиотеку (например `https://feeds.simplecast.com/54nAGcIl` для подкаста The Daily).

### Запуск в режиме разработки
Для активной разработки с hot-reload:
```bash
docker compose --profile dev up -d
```
Откройте http://localhost:5173 для работы с React через Vite.

## Внутренний процесс получения данных

Когда пользователь добавляет новую ссылку:
1. React отправляет POST запрос на `/api/podcasts/add/`, передавая `{ "url": "..." }`.
2. Apache пересылает этот запрос в контейнер Django на порт 8000.
3. Django вызывает `parse_podcast_rss()`: читает XML, извлекает название, описание и ссылку с картинкой.
4. Данные сохраняются в базу PostgreSQL с помощью модели `PodcastFeed`.
5. Frontend обновляет список карточек подкастов, показывая только что добавленный подкаст.
